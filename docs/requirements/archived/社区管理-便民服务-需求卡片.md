# 4. 社区管理（壮壮）

## 4.2 便民服务

---

### 【功能概述】

运营人员在后台管理便民服务的全流程运转。游客在 C 端可以下单选择便民服务（送货、行李搬运、垃圾清运、送水、布草配送），后续由服务人员接单、报价、上门服务，完成闭环。同时运营可管理服务人员、片区、派单配置和价格仲裁。

**涉及页面**：

| 端 | 页面 | 说明 |
|---|---|---|
| 桌面端 | 服务概览 | 各状态订单数量统计看板 |
| 桌面端 | 派单列表 | 订单列表按状态分组展示，支持手动派单 |
| 桌面端 | 片区管理 | 管理服务片区划分 |
| 桌面端 | 派单配置 | 自动派单开关及策略配置 |
| 桌面端 | 服务人员 | 服务人员列表，管理上下线 |
| 桌面端 | 价格仲裁 | 处理价格争议订单 |
| C端 | 便民服务列表 | 展示六种便民服务，选择后下单 |
| C端 | 服务追踪 | 查看订单状态流转和进度 |

**角色权限**：

| 角色 | 能做什么 |
|---|---|
| 平台运营 | 查看订单流转、手动派单、管理服务人员和片区、处理价格争议 |
| 服务人员 | 接单、报价、上门服务、确认完工 |
| 游客（C端） | 下单、支付、确认完成、评价 |

**服务类型**：

| 服务 | 计价 | 模式 |
|---|---|---|
| 生活垃圾清运 | 30元/方 起 | 定点网格 |
| 建筑垃圾清运 | 50元/方 起 | 定点网格 |
| 布草配送 | 15元/包 起 | 定点网格 |
| 送水服务 | 10元/桶 起 | 定点网格 |
| 行李搬运 | 10元/件 起 | 点对点 |
| 送货服务 | 30元/趟 起 | 点对点 |

---

### 【功能闭环流程说明】

#### 一、订单的完整生命周期

```
游客下单 → 待派单 → 已指派 → 已接单 → 已核价 → 已收款 → 服务中 → 完工待确认 → 已完成
                                                          ↓
                                                     协商中 / 取消审批中 / 待人工处理
```

#### 二、C 端下单流程

**1. 选择服务**

游客进入便民服务页，看到六种服务卡片，选择需要的服务。

**2. 填写订单**

弹出下单表单，填写：
- 地址（点对点服务还需填写目的地地址）
- 需求描述
- 图片（可选，最多 5 张）
- 期望服务时间（立即或预约）

**3. 提交订单**

下单后订单状态为"已下单"。系统会自动尝试派单。

#### 三、派单流程

**1. 自动派单（系统默认）**

系统根据服务类型自动匹配合适的服务人员：
- **定点网格类服务**（垃圾清运、布草、送水）：根据地址所在片区，找该片区任务最少的在线服务人员
- **点对点类服务**（送货、行李搬运）：根据地址距离，找最近的在线服务人员

系统依次尝试 3 位服务人员，每人有对应的接单概率：
- 有人接单 → 订单进入"已指派"
- 无人接单 → 订单停留在"待派单"，等待人工处理

**2. 手动派单（人工兜底）**

运营在后台派单列表页可以看到待派单和待人工处理的订单，点击「手动派单」选择服务人员完成指派。

#### 四、服务流程

**1. 接单与报价**

- 服务人员接单后，订单从"已指派"变为"已接单"
- 服务人员提交报价金额 → 进入"已核价"

**2. 支付**

游客在 C 端看到报价后：
- 确认支付 → 选择支付方式（微信支付或现金支付）→ 状态变为"已收款"
- 对价格有异议 → 发起价格争议 → 进入"协商中"
- 超过 15 分钟未支付 → 进入"待人工处理"

**3. 上门服务**

- 支付后服务人员开始服务 → 状态变为"服务中"
- 服务完成后 → 变为"完工待确认"
- 游客确认完成或 24 小时后自动确认 → 变为"已完成"
- 游客可对服务进行评价

#### 五、后台管理

**1. 服务概览**

看板形式展示全量订单的状态分布：
- 待派单数量（红点提示）
- 待人工处理数量
- 服务中各状态汇总

**2. 派单列表**

按状态分组展示订单，支持筛选和搜索。关键操作：

| 状态 | 运营可操作 |
|---|---|
| 待派单 / 待人工处理 | 手动派单 |
| 取消审批中 | 同意取消 / 驳回取消 |
| 协商中 | 修改报价 / 取消订单 |

**3. 服务人员管理**

- 服务人员列表：姓名、电话、状态（在线/忙碌/休息/离线）、当前单数
- 支持启用/禁用服务人员
- 可查看服务人员所负责的片区和服务类型

**4. 片区管理**

管理服务片区的基础数据（片区划分）。

**5. 派单配置**

- 自动派单的开启/关闭开关
- 查看自动派单日志（成功、失败、重试、人工操作记录）

**6. 价格仲裁**

处理进入"协商中"的订单，运营可：
- 修改报价 → 返回"已核价"状态，让游客重新支付
- 取消订单 → 进入"已取消"

#### 六、取消流程

不同状态下的取消方式：

| 当前状态 | 取消方式 |
|---|---|
| 已下单 / 待派单 | 游客可直接取消 |
| 已指派 / 已接单 / 已核价 | 游客发起取消申请 → 运营审批 |
| 协商中 | 运营可直接取消 |
| 待人工处理 | 运营可选择取消或重新派单 |
| 服务中 / 已完成 | 不可取消 |

取消审批中的订单，运营可：
- **同意取消** → 状态变为"已取消"
- **驳回取消** → 恢复到之前的状态

#### 七、超时兜底

| 场景 | 超时时间 | 处理方式 |
|---|---|---|
| 自动派单无响应 | 依次尝试 3 位 | 进入"待人工处理" |
| 游客已核价后未支付 | 15 分钟 | 进入"待人工处理" |
| 价格协商超时 | 30 分钟 | 进入"待人工处理" |
| 完工后游客未确认 | 24 小时 | 自动确认完成 |

#### 八、边界情况

| 情况 | 处理方式 |
|---|---|
| 服务人员不在线 | 不参与自动派单，跳过 |
| 价格争议 | 进入协商中，运营后台干预 |
| 多次派单失败 | 等待运营手动派单 |
| 游客超时未支付 | 订单标记为待人工处理 |
| 取消申请 | 运营审批后执行 |
