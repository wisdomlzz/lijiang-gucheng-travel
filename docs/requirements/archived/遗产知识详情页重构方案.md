# 遗产知识详情页重构方案

## 现状问题

1. 当前8种遗产类型使用统一详情页，通过 `extraFields` 动态渲染，每种类型字段差异大但共用同一组件
2. 类型命名与接口文档不一致（如 `tree` vs `ancient-tree`，`residence` vs `protected-house`）
3. 字段缺失：接口文档中的专用字段（如古桥材质/荷载、古树拉丁名/胸围、历史建筑CAD图纸等）未完整覆盖
4. 数据层与展示层耦合，不利于后续对接真实接口

## 类型映射

| 接口文档类型 | 现有类型 | 说明 |
|-------------|---------|------|
| road | road | 道路，字段不变 |
| water | water | 水系，增加传感器字段 |
| well | well | 古井，字段基本一致 |
| bridge | bridge | 古桥，增加材质/荷载字段 |
| ancient-tree | tree | 古树名木，类型改名，增加拉丁名/保护等级 |
| historic-building | publicStructure | 历史建筑，类型改名，字段大量增加 |
| protected-house | residence | 保护民居，类型改名，字段丰富 |
| cultural-relic | — | 文保单位，**新增类型** |
| human-environment | culturalEnvironment | 人文环境，类型改名 |

## 涉及文件

| 文件 | 改动 |
|------|------|
| `src/c-end/types/heritage.ts` | 类型定义对齐接口文档，补全字段 |
| `src/c-end/data/heritage/*.ts` | 每个类型独立数据文件，同步更新 |
| `src/c-end/pages/heritage/HeritageDetailPage.tsx` | 拆分为9个独立详情页组件 |
| `src/c-end/pages/heritage/` | 新增 `detail/` 目录，每个类型一个组件 |
| `src/c-end/routes.tsx` | 路由改为按类型分发 |
| `src/c-end/pages/HeritagePage.tsx` | 列表页类型标签同步更新 |

## 实施步骤

### Step 1: 对齐类型定义

```

├── index.ts                 # 路由分发

├── WaterDetail.tsx

├── AncientTreeDetail.tsx
├── HistoricBuildingDetail.tsx
├── ProtectedHouseDetail.tsx

```

### Step 3: 更新路由

```tsx
// routes.tsx
{ path: "heritage/road/:id", element: <RoadDetail /> }
{ path: "heritage/water/:id", element: <WaterDetail /> }
// ... etc
```

类型筛选标签与接口文档对齐。
