## 6. 需求点 04｜小程序资质/合规（支付、退款、提现/转账、商户/平台模式）

> 本节分两类内容：
> 1）会议中出现的“关注点/问题”（归并记录）；
> 2）官方资料摘录（便于申请/对接）。

### 6.1 会议中出现的关注点/问题（归并）

- **支付开通/材料**：会议中明确提到支付功能需要申请资质/准备资料（营业执照、法人身份证、对公账户等在讨论中出现）。
- **资金入账/提现/手续费**：会议中讨论了“交易款先到微信商户号、再提现到银行卡”的模式与手续费、报表/对账可视性问题。
- **分销/清分/分账**：
  - 会议中出现“线上分销/清分属于电商平台的一种、需要相应资质（否则违法）”的表述。
  - 同时讨论“改为线下结算（识别订单来源后线下结算给商家）”的处理方式。
- **电商平台 vs 自营**：会议中讨论了“自营”与“第三方商家入驻/平台化”的差异，以及平台化可能带来的电信/电商资质要求（会议原话为“再去咨询/查资料”）。

### 6.2 微信支付：小程序接入支付（官方材料/流程摘录）

来源：微信支付商户平台《小程序接入支付》
- https://pay.weixin.qq.com/static/applyment_guide/applyment_detail_miniapp.shtml

摘录要点：

- **企业主体申请材料**：
  - 营业执照（彩色扫描件/照片）
  - 组织机构代码证（若三证合一则无需）
  - 对公银行账户（含开户行省市信息、账号）
  - 法人身份证（彩色扫描件/照片）
- **申请流程**：提交资料 → 审核通过后签约（页面描述为 1–2 个工作日完成审核）。
- **费用**：申请接入不收取申请费用；交易按费率收取服务费（与经营类目有关）。

### 6.3 微信支付：小程序支付开发接入准备/开发流程（官方摘录）

- 开发接入准备（更新时间 2026-05-19）：
  - https://pay.weixin.qq.com/doc/v3/merchant/4015459512
- 开发指引（更新时间 2026-06-09）：
  - https://pay.weixin.qq.com/doc/v3/merchant/4012791911

摘录要点：

- 前置：小程序需公众平台完成认证，未认证无法申请支付权限。
- 绑定：商户号与小程序 AppID 需完成授权绑定。
- 基本支付链路：下单（获取 `prepay_id`）→ 小程序 `requestPayment` → 查单确认 → 支付成功回调通知 → 对账下载账单 →（需要时）退款。
- 交易类小程序规范：页面提示需满足《交易类小程序运营规范》，并按规范接入“订单发货管理功能”，否则可能被限制正式环境调起支付。

### 6.4 微信支付：退款/原路退回（官方摘录）

来源：微信支付商户文档中心《申请退款》
- https://pay.weixin.qq.com/doc/v3/merchant/4013071036

摘录要点：

- 退款时限：交易完成后 **一年内（支付成功起 +365 天）**可申请退款。
- 退款方式：接口描述为“将支付金额全部或部分 **原路退还至用户**”。

### 6.5 微信支付：转账到零钱（提现/向个人转账相关场景）（官方摘录）

来源：微信支付商户文档中心《产品介绍_商家转账到零钱》（页面内提示旧版/新版）
- https://pay.weixin.qq.com/doc/v3/merchant/4012064660

摘录要点：

- 产品定位：商户向用户微信零钱转账；支持页面/ API 发起。
- 运营账户：运营业务需通过“运营账户”出资；运营账户开通后不可关闭。
- 准入条件（页面原文）：
  - 已成为微信支付商户；当前暂不支持小微商户和个体工商户。
  - 良好的信用记录。
  - 开通申请需提交转账场景证明资料，并签订转账场景真实性承诺函。

## 7. 需求点 05｜开发运营管理系统（RCD-015：点位/上报/巡查/进销存/可视化）

来源：截图 `10_djwp.png` + 税务合规纪要中对“上报字段/巡查字段”的讨论

### 7.1 背景与总体需求（截图 1–4 点）

- 开发+运营：合作运营、自身产品销售、文创产品开发销售等；当前运营项目 12 个。
- 进销存系统：类目自动拓展；核心为数据分析（月/年对比，找高销量时间、区域等），结合 AI 简化数据录入（图片/附件上传），形成报告。
- 公共空间利用情况可视化：掌握项目运营情况（类目、位置、状态等）。
- 合作经营项目较多（如 AI 旅拍机）：支持外部系统导出数据后导入新系统，基于导入数据做统计分析。

### 7.2 需求条目（截图 5–7 点）

- 点位与基础信息展示：增删改查。
- 运营数据上报与统计分析：
  - 字段：时间（日/周/月）、经营项目、位置（分点位）、销售的产品/套餐、数量、营业额、分成比例、分成后的金额。
  - 说明：总项目名称下不同点位分开报，上传后汇总统计分析。
- 巡查：巡查记录表（客户提供），字段不固定。
  - 字段示例：设备外观、运行状态、系统、照明、灯光；餐饮可能涉及食品安全；设备可能是外观/使用情况等。
