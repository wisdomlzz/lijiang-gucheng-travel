**数据字段说明**：

- 价格是参考价,最终以师傅现场报价为准
- 生产需将硬编码价格改为从 API 读取

**数据字段说明**:

| 展示区域 | 字段（中文） | 数据字段名 | 数据来源 | 业务逻辑 |
|----------|-------------|-----------|----------|----------|
| 服务类型 | 服务名称 | `serviceType` | 用户选择,从 ServicesPage 传入 | 必填,决定派单模式 |
| 服务地址 | 地址 | `address` | 用户输入或从 addressStore 选 | 必填,派单用 lat/lng 计算距离 |
| 目标地址 | 目标地址 | `addressTo` | 用户输入 | 可选,仅点对点服务显示 |
| 备注 | 备注 | `note` | 用户输入 | 可选 |
| 图片 | 多图 | images(JSON) | 用户上传 | 可选,最多 5 张 |
| 预约时间 | 偏好时间 | `preferredTime` | 用户选择 | 尽快上门或具体时间段 |
| 经纬度 | 纬度/经度 | `lat`, `lng` | 地址选择时自动填充 | 派单引擎用 |

**业务逻辑**:
- 提交时调用 POST /api/v1/orders,参数: { serviceType, address, remark?, lat?, lng? }
- 支付方式不在下单时选择,在师傅报价后由用户确认时再锁定
- 提交按钮必须有 loading 态防止重复提交
- 成功提交后自动触发 Server 定时派单
