## 一、产品定位与边界

### 1.1 我们在做什么

古城资讯是丽江古城游 C 端的内容发布模块。游客可在 App 内浏览景区相关新闻和便民分类信息，查看信息详情；桌面端管理员可对资讯内容进行增删改查管理。

**核心目标:**

- ✅ 让游客在 App 内获取古城最新新闻和活动资讯
- ✅ 让用户按分类浏览便民信息（公房公告、房屋信息、招聘、其它）
- ✅ 让桌面端管理员便捷地管理所有资讯内容

### 1.2 两个子功能

| 子功能 | 页面入口 | 说明 |
|--------|---------|------|
| **古城新闻 (News)** | `/c/news` | 景区相关的新闻/活动资讯，带动态分类 Tab 和封面图卡片列表 |
| **便民信息 (Info)** | `/c/info` | 分类信息发布/浏览（用户可发布），含搜索、分类 Tab、加载更多 |
| **信息详情** | `/c/info/:id` | 资讯详情页，两个子功能共用 |

> **重要说明：** 两个子功能共享同一个数据模型（`NewsItem`）和同一个后端表（`content_news`），仅在 `category` 字段上做前端展示区分。

### 1.3 MVP 原则：什么必须做，什么可以等

| 优先级 | 原则 | 说明 |
|--------|------|------|
| 🔴 必须 | **浏览流程必须闭环** | 信息列表 → 详情页，路径清晰 |
| 🔴 必须 | **桌面端管理必须完整** | 管理员可 CRUD 资讯内容 |
| 🔴 必须 | **数据同步正确** | 桌面端修改后 C 端正确展示 |
| 🟡 可以简化 | 用户发布可以先不做 | "我要发布" FAB 已置，表单页面未注册路由 |
| 🟡 可以简化 | 我的发布可以先 mock | 当前为本地硬编码数据，无后端对接 |
| 🟢 以后做 | 审核流转、评论、收藏、推送 | |

### 1.4 明确不做的（当前版本边界）

- ❌ C 端信息发布页 (路由 `/c/info/create`) 未注册路由，无对应页面
- ❌ 用户发布后的审核流转流程（status pending / published / rejected 仅在前端 MyPostsPage 定义，无后端对接）
- ❌ 富文本编辑器（纯文本 + 图片）
- ❌ B 端服务人员侧的资讯管理入口
- ❌ 资讯评论/点赞/收藏
- ❌ 消息推送/订阅通知
- ❌ 资讯分类后台可配置化（分类硬编码在代码中）

## 三、核心业务流程

### 3.1 信息浏览主流程

```
用户进入 C 端
    ↓
浏览入口选择
    ├─→ 古城新闻 (/c/news)
    │       ↓
    │   动态分类 Tab 筛选（category 自动展开去重）
    │       ↓
    │   卡片列表（封面图 + 标签 + 标题 + 摘要 + 日期）
    │       ↓
    │   点击卡片 → /c/info/:id 资讯详情页
    │
    └─→ 便民信息 (/c/info)
            ↓
        4 个固定分类 Tab（公房公告/房屋信息/举贤纳仕/其它）
            ↓
        关键词搜索（按标题匹配）
            ↓
        卡片列表（图文左+字右布局），加载更多（每次 10 条）
            ↓
        点击卡片 → /c/info/:id 资讯详情页
            ↓
        【未实现】"我要发布" FAB → /c/info/create → 无路由
```

### 3.2 资讯详情页

```
进入 /c/info/:id
    ↓
头图（220px 高） + 渐变遮罩
    ↓
标题栏（返回按钮 + 分类名称）
    ↓
悬浮标题卡片（heroTitle，底部）
    ↓
标题 + 日期
    ↓
详情卡片区
    ├─ "详情" 标识 + 竖线装饰
    ├─ 副图 subImage（180px 高）
    └─ 正文段落 body（string[]）
    ↓
如 id 不匹配 → 展示 "信息不存在"
```

### 3.3 桌面端资讯管理

```
管理员登录桌面端
    ↓
内容管理 (/desktop/gates/content)
    ↓
选择 "古城资讯" Tab
    ↓
展示资讯表格（分页，每页 10 条）
    ├─→ 搜索框按标题搜索
    ├─→ 新增 → 弹窗表单 → 保存 → 列表刷新
    ├─→ 编辑 → 弹窗预填 → 修改保存 → 列表刷新
    └─→ 删除 → 确认弹窗 → 删除 → 列表刷新
```

### 3.4 我的发布（当前为 mock 数据）

```
用户进入个人中心 → "我的发布" (/c/my-posts)
    ↓
Tabs: 便民信息 (count=2)
    ↓
展示硬编码 Mock 数据列表
    ├─→ 每条记录展示：状态标签（审核中/已发布/已拒绝）+ 标题 + 正文预览 + 时间
    ├─→ 已拒记录显示拒绝原因框
    ├─→ 已发布记录显示浏览数和点赞数
    ├─→ 已拒记录可"重新提交"（仅 toast 反馈）
    └─→ 所有记录可"删除"（仅本地状态删除）
```

## 五、核心数据模型

### 5.1 NewsItem（内容资讯）

> 两个子功能共享同一数据模型和 `content_news` 数据表。桌面端通过"古城资讯"标签页统一管理。

```typescript
interface NewsItem {
  id: string
  imageUrl: string          // 列表封面图 URL
  title: string             // 资讯标题
  tag: string               // 标签文案 (如 "热点"、"公告")
  tagColor: string          // 标签颜色 hex (如 "#3B82F6")
  date: string              // 发布日期 (YYYY-MM-DD)
  summary: string           // 摘要文本（列表展示）
  category: NewsCategory    // 分类
  heroTitle?: string        // 详情页悬浮卡片标题
  body?: string[]           // 详情页正文段落数组
  subImage?: string         // 详情页正文配图 URL
}

type NewsCategory = "公房公告" | "房屋信息" | "举贤纳仕" | "其它"
```

### 5.2 Post（用户发布，当前 mock）

> MyPostsPage 内定义的本地类型，无真实后端表对应，无持久化。

```typescript
interface Post {
  id: string
  type: "info"              // 发布类型（当前仅为 info）
  title: string
  content: string
  status: "pending" | "published" | "rejected"
  rejectReason?: string
  views: number
  likes: number
  createdAt: string
}
```

### 5.3 后端表

| 表名 | 说明 |
|------|------|
| `content_news` | 资讯主表，通过 `crudRoutes()` 自动生成 RESTful 路由 |

### 7.1 信息浏览

- [✅] 古城新闻页 (`/c/news`) 展示资讯卡片列表，每张含封面图、标签、标题、摘要、日期

- [✅] 便民信息页 (`/c/info`) 展示 4 个固定分类 Tab
- [✅] 便民信息可按分类切换展示
- [✅] 便民信息可按标题关键词搜索
- [✅] 列表支持"加载更多"分页（每次 10 条）
- [✅] 点击列表卡片跳转到详情页 (`/c/info/:id`)

- [✅] 详情页标题栏右端留白保持标题居中
- [✅] 资讯不存在时显示"信息不存在"空态
- [✅] 列表无数据时显示"暂无相关信息"空态

### 7.2 桌面端管理

- [✅] 内容管理页 `NewsManageContent` 以表格列表展示所有资讯
- [✅] 支持按标题搜索
- [✅] 支持分页（每页 10 条）

- [✅] 桌面端新增/编辑/删除的内容在 C 端列表和详情页正确同步
- [✅] 页面刷新后数据不丢失（hydration 从后端 API 加载）
