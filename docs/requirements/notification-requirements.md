# 消息通知（Notification）

## 产品定位与边界

消息通知模块提供丽江古城游 C 端用户的站内消息收件箱功能。接收订单、活动、系统三类通知，支持标记已读、删除、按类型筛选，并模拟实时推送效果。

边界：
- 仅限 C 端（游客端）访问
- 无桌面端后台管理通知
- 实时推送为前端模拟（定时器），非真实 WebSocket/SSE

## 核心用户角色

| 角色 | 描述 |
|------|------|
| 游客（tourist） | 接收和查看消息通知 |

## 核心业务流程

1. 用户进入消息通知页
2. 默认展示全部通知列表，未读通知以左边框高亮标识
3. 支持按类型 Tab（全部/订单/活动/系统）筛选
4. 点击通知标记已读并跳转目标页面
5. 左滑显示删除按钮
6. 支持一键全部标记已读

## 核心数据模型

```typescript
type NotificationType = "order" | "activity" | "system"

interface Notification {
  id: string
  type: NotificationType
  title: string
  summary: string
  time: string
  isRead: boolean
  targetUrl?: string
  createdAt: string
}
```

通知 Store 定义在 `platform/notification/`（平台级基础设施，非 feature 级别）。
