## 一、产品定位与边界

### 1.1 我们在做什么

签到打卡 feature 包含**三个独立子系统**，定位不同但共享 C 端游客身份：

| 子系统 | 一句话定位 | 目标 |
|---|---|---|
| 文化院落打卡 | 游客到达文化院落，GPS 定位打卡 | 积累游览足迹 + 积分激励 |
| 纳西人打卡 | "今天我想做纳西人"连续签到挑战 | 文化体验 Gamification + 积分激励 |
| 公众随手拍上报 | 游客上报古城遗产要素异常（地砖松动/占道/垃圾等） | 公众参与遗产保护 |

### 1.2 MVP 原则

| 优先级 | 原则 | 说明 |
|---|---|---|
| 红色 必须 | **主流程必须闭环** | 打卡→记录→展示→积分，每一步不断 |
| 红色 必须 | **数据持久化** | 打卡记录通过 API 存入服务端 |
| 黄色 可简化 | 拍照用占位图 | Demo 不需要真实相机调用 |
| 黄色 可简化 | GPS 用模拟定位 | Demo 阶段不做真实定位 |
| 绿色 以后做 | 补签机制、成就系统、排行榜社交等 |

### 1.3 明确不做的（Demo 范围外）

- 成就/徽章/集章系统
- 打卡排行榜 C 端展示（仅桌面端管理后台有排行）
- 手机相册直接调起拍照（仅文件上传）
- 随手拍后台审核流程（状态展示但后台无审核页面）
- 随手拍积分奖励
- 定位 LBS 服务接入真实地图 API
- 补签卡/连续打卡中断补救
- 打卡照片的人工审核与删除
- 文化院落打卡服务器端种子数据（仅在客户端 store 中管理）

## 三、核心业务流程

### 3.1 文化院落打卡流程

```
游客进入 C 端文化院落列表
    ↓
浏览院落，选择感兴趣的文化院落
    ↓
进入院落详情页
    ↓ （自动模拟 GPS 定位）
系统模拟计算当前位置与院落距离
    ↓
    ├─ 距离 < 500m → 显示"去打卡"按钮（可点击）
    └─ 距离 > 500m → 按钮置灰，提示"超出打卡范围"
    ↓
点击"去打卡"
    ↓
系统校验：userId + courtyardId + 今日是否已打卡
    ├─ 今日已打卡 → 提示"今日已打卡"，不创建记录
    └─ 未打卡 → 创建打卡记录，调用积分接口 += 5 分
    ↓
流程结束，打卡记录写入 store 并持久化到服务端
```

**自定义规则**：
- 每日同人同院落仅 1 次（`userId + courtyardId + date` 三重校验）
- 隔天可重新打卡
- 不限院落数（一天内可打卡多个院落）
- 统计口径为"打卡次数（人次）"，同一人不同日打卡 N 次，榜单计 N 次

### 3.2 纳西人连续打卡流程

```
游客进入 C 端"我的" → "今天我想做纳西人"
    ↓
查看连续打卡天数 + 7 天进度条 + 累计打卡天数
    ↓
穿纳西族服饰拍照（模拟拍照 / 上传照片）
    ↓
系统校验：今日是否已打卡
    ├─ 已打卡 → 提示"今日已打卡"
    └─ 未打卡 → 创建打卡记录
    ↓
计算新的连续天数
    ├─ 连续天数 % 7 == 0 → 触发积分奖励 += 50 分（naxi_streak）
    └─ 未达成 7 天 → 仅记录打卡
    ↓
流程结束，跳转到积分页面展示
```

**连续天数计算规则**：
- 从今天或昨天开始倒推连续日期
- 如果今天打卡了且昨天也打卡了 → 连续天数递增
- 如果今天没打卡 → 检查是否昨天打卡了（是则从昨天开始倒推）
- 如果昨天也没打卡 → 连续天数为 0
- 每 7 天为一个奖励周期，触发积分奖励（7/14/21...）
- **无补签机制**：中间断一天，连续天数重置

### 3.3 随手拍上报流程

```
游客进入 C 端"随手拍记录" → 点击右下角"+"按钮
    ↓
填写上报表单：
    ├─ 片区（古城/白沙/束河）→ 级联
    ├─ 类型（历史街巷/水系/古井/桥梁/...）→ 级联
    ├─ 对象（四方街/卖草场/新华街/...）→ 级联
    ├─ 位置（点击定位模拟获取）
    ├─ 详细地址（手动输入）
    ├─ 异常情况描述（文本）
    └─ 照片（上传，最多 9 张）
    ↓
表单校验 → 提交 → 显示成功弹窗
    ↓
提交的记录出现在"随手拍记录"列表页
    ↓
管理员（模拟）审核
    ├─ 已核实（processed）
    └─ 驳回（rejected）
```

## 九、架构说明

### 9.1 打卡防刷规则

同一个人（userId）在同一天内对一个院落（courtyardId）最多只允许打卡 1 次。

```
每日限制 = userId + courtyardId + date（精度到日）
重复打卡 → 提示"今日已打卡"，不创建记录
隔天可重新打卡
```

### 9.2 桌面端统计口径

| 指标 | 口径 |
|---|---|
| 总打卡次数 | 时间段内的记录总数 |
| 参与游客数 | 时间段内的去重 userId 计数 |
| 院落排行 | 按 courtyardId 分组，打卡次数降序 |
| Top 默认展示 | 5 个，点击"展开全部"展示所有 |

### 9.3 纳西打卡连续天数计算

```
从当天或前一天开始，倒推连续日期。
如果今天打卡了 → streak 至少为 1，继续向前推
如果今天没打卡但昨天打卡了 → 从昨天开始 streak 至少为 1
如果昨天也没打卡 → streak = 0
```
