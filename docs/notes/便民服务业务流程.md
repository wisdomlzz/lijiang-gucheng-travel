## 1. 服务类型

| 类型 | 分类 | 特点 |
| --- | --- | --- |
| 送货服务 | 点对点 | 固定起止地址 |
| 行李搬运 | 点对点 | 固定起止地址 |
| 建筑垃圾清运 | 片区型 | 按片区匹配人员 |
| 生活垃圾清运 | 片区型 | 按片区匹配人员 |

## 2. 三端角色

| 端 | 角色 | 核心操作 |
|---|---|---|
| C端（游客/商户） | 下单方 | 下单、付款、取消申请、确认完工、评价、投诉 |
| B端（服务人员） | 服务方 | 接单、报价、开始服务、完工拍照、同意/拒绝取消 |
| 桌面端（管理员） | 平台方 | 手动派单、取消审批、重新派单、处理异常 |

## 4. 正常流程详解

### 4.1 C端下单（S10）

C端用户选择服务类型，填写：
- 地址（点对点：起送+送达地址；片区型：服务地址）
- 图片（可选）
- 备注
- 期望时间

下单后状态 → **S10 已下单**，500ms 后自动派单。

### 4.2 派单（A10 → A20）

**自动派单（默认）：**
1. S10 → A10（待派单）
2. 按服务类型匹配合适的服务人员
3. 按距离/片区匹配在线人员 → A20（已指派）

**手动派单（桌面端）：**
- 管理员选择指定人员 → 直接到 A20

### 4.3 B端服务流程

| 步骤 | 操作 | 状态 |
|---|---|---|
| 1. 接单 | B端查看指派订单，点击"接单" | A20 → A30 |
| 2. 报价 | B端填写服务价格 | A30 → A35 |
| 3. 收款 | C端付款（在线/现金） | A35 → A40 |
| 4. 开始服务 | B端点击"开始服务" | A40 → S48 |
| 5. 完工拍照 | B端上传完工照片 | S48 → S55 |
| 6. 确认完工 | C端确认 或 30s自动确认 | S55 → S40 ✅ |

### 4.4 评价

订单完成后，C端可进行 1-5 星评价。

### 4.5 跨域联动

订单完成（S40）→ 自动录入结算收入（`useSettlementStore.recordIncome`）

## 5. 异常流程详解

### 5.1 支付超时（A35 → S90）

```
报价后 15s 未付款
    A35 已核价 ──→ S90 待人工处理
                       │
              ┌────────┴────────┐
              ▼                 ▼
         reDispatch          forceCancel
         重新派单 → A10       强制取消 → S50
```

### 5.2 取消订单

取消流程使用**订单标记（cancelRequested）** 而非独立状态码：

| 当前状态 | 取消方式 | 结果 |
|---|---|---|
| S10 / A10（未指派） | 直接取消 | → S50 |
| A20~S55（已指派） | 设置 cancelRequested=true | 保持原状态，通知管理员 |
| 已标记 | 管理员同意 | → S50 |
| 已标记 | 管理员拒绝 | cancelRequested=false，继续服务 |
| S90 | 管理员强制取消 | → S50 |

注意：**砍掉了原有的 R80 独立状态码**。取消申请不再是一个状态转换，而是订单上的一个布尔标记。管理员在桌面端「取消审批」页或「便民服务管理→派单管理」中处理。

### 5.3 投诉

订单任意状态 → 创建投诉 → C10（已提交）→ C40（已处理）/ CR（已驳回）

## 6. 业务规则

### 6.1 付款规则
- 报价后 15s 内完成支付，否则自动进入 S90（演示用）
- 支持在线支付和现金支付

### 6.2 派单规则
- **点对点服务**：按服务类型匹配 + 按距离排序（最近优先）
- **片区型服务**：按服务类型 + 片区（zoneId）严格匹配
- 管理员可手动指定人员

### 6.3 确认规则
- C端手动确认完工
- 30s 未确认则自动确认（演示用）

### 6.4 通知规则
- 每个状态转换自动触发通知推送到对应端
- 派单通知 → B端（服务人员）
- 核价/完工通知 → C端（下单用户）
- 取消申请通知 → 桌面端（管理员）

## 7. 定时器清单

| 定时器 | 触发条件 | 动作 |
|---|---|---|
| 下单后500ms | 创建订单 | S10→A10→A20（自动派单） |
| `conv:{id}:pay` | A35报价后15s | A35→S90（未付款） |
| `conv:{id}:autoConfirm` | S55完工后30s | S55→S40（自动确认） |

## 8. 状态码一览

| 状态码 | 标签 | 阶段 | 说明 |
|---|---|---|---|
| S10 | 已下单 | 初始 | 用户提交订单 |
| A10 | 待派单 | 进行中 | 等待指派服务人员 |
| A20 | 已指派 | 进行中 | 已指定服务人员 |
| A30 | 已接单 | 进行中 | 服务人员已接单 |
| A35 | 已核价 | 进行中 | 服务人员已报价 |
| A40 | 已收款 | 进行中 | 用户已付款 |
| S48 | 服务中 | 进行中 | 服务人员开始服务 |
| S55 | 完工待确认 | 终态前 | 服务人员上传完工照片 |
| S40 | 已完成 | ✅终态 | 双方确认完成 |
| S50 | 已取消 | ⚠️终态 | 订单取消 |
| S90 | 待人工处理 | ⚠️ | 异常状态，需人工介入 |
