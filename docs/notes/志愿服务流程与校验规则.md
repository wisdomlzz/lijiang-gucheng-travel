# 志愿服务 —— 流程与校验规则

## 一、三条状态机

### 1.1 志愿者状态

```
未注册 ──注册──► pending(待审核) ──通过──► approved(已通过)
                                  └──驳回──► rejected(已驳回) ──重新上传资质──► pending
```

- `register()`：游客提交资料，自动进入 `pending`
- `approveVolunteer()`：审核通过，状态变 `approved`
- `rejectVolunteer(id, reason)`：驳回，需填写驳回原因
- `resubmitVolunteer(id, images)`：重新上传资质后回到 `pending`，审核历史保留不丢失

### 1.2 活动状态

```
draft(草稿) ──publish──► published(已发布)
                            │
                       (到达开始时间)
                            ▼
                     in_progress(进行中)
                            │
                       (到达结束时间/强制结束)
                            ▼
                        ended(已结束)

  任意非终态 ──cancel──► cancelled(已取消)
```

- 活动无需审核，创建后可直接发布
- **自动流转**：到达开始时间自动转为 `in_progress`，到达结束时间自动转为 `ended` 并结算异常
- `forceEndActivity()`：强制结束，同步触发异常结算

### 1.3 签到状态

```
signed_up(已报名) ──checkIn──► checked_in(已签到) ──checkOut──► checked_out(已签退)

  活动结束未签到 ───── auto ──► no_show(缺席)
  活动结束未签退 ───── auto ──► checkout_overdue(未签退) ──resolveAbnormal──► checked_out(管理员补录)
```

## 二、校验规则

### 2.1 报名校验

| 规则 | 说明 |
|------|------|
| 报名时间窗口 | 当前时间需在「报名开始 ~ 报名截止」区间内 |
| 名额上限 | 报名人数 < 最大参与人数 |
| 重复报名 | 同一志愿者不能重复报名同一活动 |
| 时间冲突 | 与已报名的其他活动时段重叠则拒绝 |
| 活动状态 | 仅 `published` / `in_progress` 可报名 |
| 登记报名开始（可选） | 设置了 `enrollStartTime` 的，到时间后才可报名 |

### 2.2 签到校验

| 规则 | 说明 |
|------|------|
| 签到窗口 | 活动开始前 30 分钟即可签到，至活动结束前 |
| 前置条件 | 仅 `signed_up` 可签到 |
| 迟到判定 | 签到时间晚于活动开始时间 30 分钟，标记为迟到并记录迟到分钟数 |
| 定位校验 | 校验志愿者在活动地点半径范围内 |

### 2.3 签退校验

| 规则 | 说明 |
|------|------|
| 前置条件 | 仅 `checked_in` 可签退 |
| 时长计算 | 签退时间 − 签到时间，不足 0.5 小时按 0.5 小时计 |
| 积分发放 | 服务时长按 `1 小时 = 2 积分` 折算，写入积分账户 |

### 2.4 表单校验

| 字段 | 规则 |
|------|------|
| 姓名 | 必填，真实姓名 |
| 电话 | 必填，11 位手机号 |
| 政治面貌 | 必填，中共党员 / 共青团员 / 群众 / 其他 |
| 工作单位 | 必填 |
| 资质图片 | 最多 5 张，单张 ≤5MB |

### 2.5 异常处理

| 异常状态 | 触发条件 | 处理方式 |
|----------|----------|----------|
| `no_show` | 活动结束时状态仍为 `signed_up` | 后台人工核实，可标记完成（0 时长）或维持缺席 |
| `checkout_overdue` | 活动结束时状态仍为 `checked_in` | 后台补录服务时长，标记「管理员补录」 |

后台处理后状态转为 `checked_out`，记录 `isManual` 标记和 `resolvedAt`；若补录时长 > 0 则同步发放积分。

## 三、自动流转机制

活动发布后注册定时器：

```
published → (到达 startTime) → in_progress → (到达 endTime) → ended → settleActivity()
```

- `settleActivity()` 处理活动结束时的异常结算：
  - `signed_up` → `no_show`
  - `checked_in` → `checkout_overdue`
- `forceEndActivity()` 也会触发结算
- 活动被取消时清除对应定时器

## 四、角色与权限

| 角色 | 端 | 能力范围 |
|------|-----|----------|
| 游客（志愿者） | C 端 | 认证、浏览活动、报名、签到签退、查看我的活动 |
| 平台管理员 | 桌面端 | 审核志愿者、创建/管理活动、处理异常签到 |

B 端（便民服务人员）不参与志愿服务流程。

## 五、C 端交互说明

### 5.1 认证页（/c/volunteer）

| 状态 | 展示 | 可执行操作 |
|------|------|-----------|
| 未注册 | 注册表单 | 填写资料+上传资质 → 提交 |
| `pending` | 提交信息卡片 + 审核中提示 | 等待审核 |
| `rejected` | 驳回原因 + 重新上传资质入口 | 重新上传 → 重新提交 |
| `approved` | 自动跳转活动列表 | — |

### 5.2 活动列表（/c/volunteer/activities）

- **发现流**：展示所有可报名活动（`published` + `in_progress`），支持搜索
- **活动卡片**：标题、地点、时间、报名进度条；已报名活动标题前有蓝点标记
- **底部弹窗（我的活动）**：展示所有报名记录，按操作优先级排序：**待签退(0) > 待签到(1) > 已报名等待(2) > 已完成(3) > 异常(4)**
- **浮动按钮（FAB）**：右下角常驻，标注「我的活动」+ 数量角标
- **自动弹出**：进入页面若有进行中/待开始的已报名活动，自动弹出一次

### 5.3 活动详情（/c/volunteer/activities/:id）

- 展示活动完整信息、报名进度、我的参与状态
- 底部按钮根据状态动态变化：

| 签到状态 | 活动时段 | 按钮展示 |
|----------|----------|---------|
| 未报名 | — | 立即报名 / 名额已满 / 报名已截止 |
| `signed_up` | 未到签到窗口 | 等待活动开始 |
| `signed_up` | 签到窗口已开 | 签到 |
| `checked_in` | 进行中 | 签退 |
| `checked_in` | 已结束 | 活动已结束，请及时签退 |
| `checked_out` | — | 已完成服务（展示时长） |
| `no_show` | — | 您未签到参加此活动 |
| `checkout_overdue` | — | 您未签退，请联系管理员处理 |

## 六、桌面端管理功能

### 6.1 志愿者审核

- 列表展示所有志愿者，支持按状态筛选
- 待审核行高亮，提供「通过 / 驳回」操作（驳回需填原因）
- 可查看资质图片大图
- 审核记录：每位志愿者的通过/驳回/重新提交历史均被完整记录

### 6.2 活动管理

- 列表展示所有活动，按 5 种状态筛选
- 每行根据活动状态提供对应生命周期操作：

| 活动状态 | 可执行操作 |
|----------|-----------|
| `draft` | 编辑、发布、删除 |
| `published` | 取消（无报名时）、查看报名 |
| `in_progress` | 强制结束、查看报名 |
| `ended` / `cancelled` | 仅查看 |

- 点击活动进入详情，查看报名列表及签到状态，处理异常记录

### 6.3 概览统计

展示 6 项统计数据：志愿者总数、待审核数、活动总数、异常待处理数、总报名人次、总服务时长。

## 七、签名与计分

- 签退或管理员补录时，按服务时长折算积分写入积分账户
- 积分公式：`Math.round(serviceHours × 2)`
- 积分变更需关联 `transactionType: "volunteer_service"` 供审计追溯
