# 4. 社区管理（壮壮）

## 4.1 公告下发

---

### 【功能概述】

运营人员在后台创建公告草稿，草稿发布后对 C 端游客可见。发布后的公告在 C 端「公告通知」宫格展示，同时首页「景区综合资讯」区域也展示最近公告。

所有公告统一为立即发布，不设置有效期，长期有效。

**涉及页面**：

| 端 | 页面 | 说明 |
|---|---|---|
| 桌面端 | 公告下发管理 | 运营管理菜单进入，管理公告的增删改查与发布/下架 |
| C端 | 公告通知列表 | 首页宫格「公告通知」进入，展示所有已发布公告 |
| C端 | 公告详情 | 列表页点击进入 |

**角色权限**：

| 角色 | 能做什么 |
|---|---|
| 平台运营 | 公告的增删改查、发布、下架 |
| 游客（C端） | 查看已发布的公告 |

---

### 【功能闭环流程说明】

#### 一、公告的完整生命周期

```
新增草稿 → 保存为"草稿" → 点击「发布」 → 游客可见
                                         ↓
                                   点击「下架」
                                         ↓
                                    游客不可见
                                         ↓
                                   点击「发布」
                                         ↓
                                    游客再次可见
```

#### 二、后台操作流程

**1. 新增公告**

运营点击「新增公告」按钮，弹窗填写：
- 标题（必填）
- 内容（必填）
- 图片（可选，最多 3 张）

点击保存后，生成一条"草稿"状态的公告，此时游客还看不到。

**2. 编辑公告**

任意状态的公告都可以点击「编辑」修改标题、内容、图片。编辑不影响公告当前的状态和发布时间。

**3. 发布**

- 草稿 → 发布 → 状态变为"已发布"，记录发布时间
- 已下架 → 发布 → 状态变为"已发布"，发布时间更新为当前时间

发布后公告对 C 端游客可见。

**4. 下架**

已发布 → 下架 → 状态变为"已下架" → 游客不可见。发布时间保持不变。

**5. 删除**

点击「删除」→ 确认后永久删除，不可恢复。

**发布时间说明**：

| 操作 | 发布时间 |
|---|---|
| 新增草稿 | 无 |
| 首次发布（草稿→已发布） | 记录当前时间 |
| 下架（已发布→已下架） | 原发布时间不变 |
| 重新发布（已下架→已发布） | 更新为当前时间 |

#### 三、列表页操作

**筛选与搜索**：
- 状态筛选：全部 / 已发布 / 已下架 / 草稿
- 标题搜索：输入关键字搜索

**各状态可操作按钮**：

| 状态 | 操作按钮 |
|---|---|
| 草稿 | 发布、编辑、删除 |
| 已发布 | 下架、编辑、删除 |
| 已下架 | 发布、编辑、删除 |

#### 四、C 端游客看到的

**1. 公告通知列表页**

只展示已发布的公告，按发布时间倒序排列：
- 支持搜索标题
- 每次加载 10 条，滑到底部自动加载更多
- 列表项展示：图片、标题、摘要、日期
- 空状态：暂无公告；搜索无结果：未找到相关公告

**2. 公告详情页**

展示：横幅图片、公告标签、发布时间、标题、正文、多张图片。

如果公告不存在或已下架，显示「公告不存在或已下架」。

**3. 首页聚合展示（关联说明）**

首页「景区综合资讯」区域聚合展示公告、古城资讯、便民信息三类内容的最近信息流。本条功能产生的已发布公告会在这里自然出现。

#### 五、边界情况

| 情况 | 处理方式 |
|---|---|
| 标题或内容为空 | 提示不能为空，禁止保存 |
| 公告不存在或已下架 | 显示"公告不存在或已下架" |
| 图片超过 3 张 | 上传按钮自动隐藏 |
| 图片格式不对或过大 | 提示具体错误 |
| 搜索无结果 | 显示"未找到相关公告" |
