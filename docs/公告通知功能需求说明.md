## 1. 功能概述

运营人员可在后台创建公告草稿，草稿经发布后对 C 端游客可见。公告发布后在 C 端「公告通知」宫格展示，同时首页「景区综合资讯」区域展示最近公告。不区分发布方式，统一为立即发布；不设置有效期，统一长期有效。

### 1.1 角色与权限

| 角色 | 端 | 操作范围 |
|------|------|---------|
| 平台运营（role_admin） | 业务应用-社区管理 | 公告 CRUD、发布、下架 |
| 游客（未登录/已登录） | C端小程序 | 查看已发布的公告 |

### 1.2 页面入口

| 页面 | 所属模块 | 来源 |
|------|---------|------|
| 公告下发管理 | 业务应用-社区管理 | 运营管理菜单 |
| 公告通知列表 | C端小程序 | 首页宫格「公告通知」 |
| 公告详情 | C端小程序 | 公告通知列表页点击跳转 |

### 1.3 首页聚合展示

C端首页「景区综合资讯」区域展示已发布公告的前 4 条。

## 2. 业务应用-社区管理 · 公告下发管理

运营人员在业务应用社区管理中，创建和管理公告。支持新增草稿、编辑、发布、下架、删除。

### 2.1 列表页

**所属**: 运营管理 → 公告下发管理  
**路由**: `/desktop/announcements`

**页面功能**:
- 以表格展示全部公告，按 `updatedAt` 倒序排列
- 状态筛选：全部 / 已发布 / 已下架 / 草稿
- 标题搜索：过滤 `title.includes(keyword)`
- 每行操作：发布/下架、编辑、删除（均为文字按钮）

**表格列定义**:

| 列头 | 数据字段 | 显示格式 |
|------|---------|---------|
| 标题 | title | 文本 |
| 状态 | status | Badge：已发布(绿) / 已下架(黄) / 草稿(灰) |
| 创建时间 | createdAt | `yyyy-MM-dd HH:mm` |
| 最近一次发布时间 | publishTime | `yyyy-MM-dd HH:mm`；草稿/下架态显示 `-` |
| 操作 | — | 文字按钮组 |

**操作按钮规则**:

| 状态 | 可见按钮 |
|------|---------|
| 草稿 | 发布、编辑、删除 |
| 已发布 | 下架、编辑、删除 |
| 已下架 | 发布、编辑、删除 |

**状态流转**:

```
草稿 ──发布──▶ 已发布 ──下架──▶ 已下架
                 ▲                │
                 └────重新发布──────┘
```

**最近一次发布时间的业务规则**:

| 操作 | publishTime 行为 | 说明 |
|------|-----------------|------|
| 新增草稿 | 空字符串 `""` | 未发布状态，无发布时间 |
| 首次发布（草稿→已发布） | 写入当前时间 | 开始对C端可见 |
| 下架（已发布→已下架） | 保持不变 | 保留历史发布时间记录 |
| 重新发布（已下架→已发布） | **更新为当前时间** | C端看到的是最近一次发布时间 |

### 2.2 新增/编辑弹窗

**触发**: 点击「新增公告」或「编辑」按钮

**表单字段**:

| 表单项 | 组件类型 | 必填 | 校验规则 |
|--------|---------|------|---------|
| 标题 | Input | 是 | 不能为空 |
| 内容 | Textarea | 是 | 不能为空 |
| 图片 | 图片上传 | 否 | 最多3张；格式 JPG/PNG/WebP；大小 ≤2MB；base64 编码 |

**保存逻辑**:
- **新增**：`status` 固定为 `draft`，`createdAt` 写当前时间，`publishTime` 置空
- **编辑**：保留原有 `status`，更新除 `id`/`createdAt` 外的所有字段，`updatedAt` 更新为当前时间
- 校验不通过：toast 提示具体错误
- 保存后关闭弹窗

### 2.3 删除逻辑

物理删除，不可恢复。

## 3. C端小程序 · 公告展示

C端小程序支持公告接收、公告列表、公告查看。

### 3.1 公告通知列表页

**URL**: `/c/notice`

**数据获取条件**:
- `status === "published"`
- 按 `publishTime` 倒序排列

**功能**:
- 标题搜索：过滤 `title.includes(keyword)`
- 每次加载 10 条，点击「加载更多」追加

**列表项 UI**:

| 区域 | 数据 |
|------|------|
| 左侧图片 | `images[0]`；无图时显示 📢 占位 |
| 标题 | title，2行截断 |
| 摘要 | content，2行截断，60字后加... |
| 底部日期 | publishTime，`yyyy-MM-dd` 格式 |

**空状态**:
- 无数据：`暂无公告`
- 搜索无结果：`未找到相关公告`

### 3.2 公告详情页

**URL**: `/c/announcement/:id`

**数据获取**: 根据 id 获取公告，校验 `status === "published"`

**页面结构**:

```
┌─────────────────────┐
│  Banner (images[0]) │  ← 无图则显示默认渐变背景
├─────────────────────┤
│  ← 返回按钮          │  ← 有图时 absolute 浮动
├─────────────────────┤
│  公告                 │
│  发布于 yyyy-MM-dd   │
│  标题 (20px, 粗体)    │
│  ───── 分割线 ─────   │
│  正文 (15px, 行高1.7) │
│  更多图片(如有多张)     │
└─────────────────────┘
```

**不存在态**: 显示「公告不存在或已下架」

## 4. 数据模型

### 4.1 Announcement

| 字段 | TS类型 | 说明 |
|------|--------|------|
| id | string | 主键，格式 `ann-{timestamp}` |
| title | string | 公告标题 [必填] |
| content | string | 公告正文，纯文本 [必填] |
| images | string[] | 图片 base64 数组，≤3 |
| type | `"公告"` | 固定值，仅管理公告 |
| publishTime | string (ISO) | 发布时间；草稿态为空字符串 |
| status | `"draft" \| "published" \| "unpublished"` | 状态 |
| createdAt | string (ISO) | 自动，创建草稿时写入 |
| updatedAt | string (ISO) | 自动，每次更新时同步 |

### 4.2 创建时间与发布时间

- `createdAt`：记录草稿的创建时间，任何时候不覆盖
- `publishTime`：仅在「发布」操作时写入当前时间；草稿和已下架态保持空字符串
- 表格列表同时展示两列（创建时间、发布时间），C端只展示 `publishTime`
- 已下架公告再次发布时，`publishTime` 更新为新的发布时间

## 7. FAQ / 注意事项

**Q: 已发布的公告可以编辑吗？**
当前支持直接编辑，编辑保存后 `updatedAt` 更新。后续版本可限制已发布公告需先下架再编辑。

**Q: 删除公告后数据还能恢复吗？**
当前为物理删除，不可恢复。

**Q: 后续对接真实接口，字段是否会有变化？**
字段结构应保持不变，主要变化是：
- 图片从 base64 改为文件上传
- 后端应提供分页接口取代前端 useLoadMore

**Q: 下架后重新发布，发布时间怎么算？**
重新发布时 `publishTime` 更新为当前时间。C端以最新发布时间排序展示。
