# 丽江古城游新版完整需求说明书

> 版本口径：`markdown_exports` 为老版本功能基线，`demo` 为新版三端演示。本文按“复用功能优先说明、P0 本期新增与优化、P1 全新便民服务全流程”组织，避免把旧版成熟能力误判为新版缺口。
>
> 三端范围：C 端小程序、B 端便民服务人员端、Desktop 管理后台。
>
> 商城边界：商品、商城订单、核销、物流、退款、商城派单、商城服务人员、SKU 派单配置均不纳入本期 P0/P1 便民服务范围。供应商入驻属于本期需求；审核通过后，后续通过平台与商城单点登录对接，由供应商进入商城后台管理自己的商品、订单、售后等。

## 1. 复用功能清单

本节先列“复用功能”。这些功能已在老版本文档中明确存在，本期新版 demo 不应逐项重做完整页面，但菜单、数据来源和业务口径必须与老文档保持一致。

## 2. P0 新增与优化功能

P0 目标：在不重做老版成熟功能的前提下，把本期必须展示的新口径闭环跑通。重点是 C 端体验收敛、Desktop 菜单与旧文档匹配、供应商入驻边界、轻量文化院落打卡、投诉优化，以及“一键下单”。

### 2.1 C 端首页与入口收敛

| 功能 | C 端要求 | Desktop 要求 | 验收点 |
|---|---|---|---|
| 首页宫格 | 保留一键服务、导览地图、文化院落、志愿服务占位、讲解服务、遗产知识、VR 游览、公房服务、官方商城、购在古城、门票预定、古城资讯、精选路线、停车服务、便民信息、一键投诉、联票套餐、随手拍 | 当前按首页配置数据展示；广告位维护只负责首页广告图，不再混入宫格排序 | “购在古城”图标存在；志愿服务进入占位页，文案为“需求待澄明” |
| 首页底部资讯 | 收敛为古城资讯 + 便民信息聚合 | 内容来自景区资讯/便民信息审核 | 不展示未定口径的志愿招募 |
| 商城入口 | 官方商城、门票、联票、讲解等外链到商城/外部系统 | Desktop 只保留商城管理后台入口 | 不进入平台侧商城派单或商城订单管理 |

### 2.2 供应商入驻与商城后台边界

| 功能 | C 端 | Desktop | 验收点 |
|---|---|---|---|
| 入驻申请 | 供应商填写主体信息、经营分类、联系人、资质材料，提交后查看状态 | 供应商入驻审核列表、详情、通过、驳回 | 供应商入驻属于本期需求 |
| 审核通过 | C 端展示审核状态 | 平台管理员审核通过后，为后续 SSO 建立供应商身份 | 商品、订单、核销、售后不在平台后台管理 |
| 供应商登录 | 不在平台桌面端管理商品订单 | 供应商在桌面端应直接进入商城后台 | 桌面端“供应商”账号点击后打开商城后台 |

### 2.5 文化院落轻量打卡

### 2.6 投诉优化

### 2.7 P0 一键下单：便民服务沿用老版基础能力

P0 的“一键下单”是轻量闭环：游客能提交便民服务订单，后台能看到并派单，用户能查看订单状态。它基于老版“便民信息/便民服务”口径扩展，但 P0 不要求完整 P1 的价格仲裁、服务人员端全流程和自动化治理全部上线。

| 环节 | C 端小程序 | Desktop 管理后台 | 验收点 |
|---|---|---|---|
| 服务选择 | 一键服务入口，选择生活垃圾清运、建筑垃圾清运、布草配送、送水服务、行李搬运、送货服务 | 服务类型保留配置基础 | 服务类型清晰，商城商品服务不混入 |
| 下单表单 | 服务地址/起终点地址、订单描述、现场照片、即刻/预约时间 | 后台生成订单记录 | 必填校验：描述、地址、照片 |
| 订单提交 | 生成便民服务订单号，进入订单详情 | 派单列表出现订单 | C 端订单只展示便民服务订单 |
| 轻量派单 | 用户可看到待派单/已指派状态 | 可手动指派服务人员 | 派单属于便民服务，不是商城派单 |
| 状态查看 | 订单列表按全部/进行中/已完成/已取消筛选；订单详情展示状态进度 | 后台查看订单状态、服务人员、地址、类型 | 订单状态不出现商城订单 |
| 基础取消 | 下单/待派单阶段可取消，已指派后可申请取消 | 后台可处理取消申请 | 取消规则有明确状态边界 |

## 3. P1 全新版本便民服务全流程

P1 目标：把便民服务从“下单入口”升级为 C 端下单与支付、B 端服务人员接单履约、Desktop 平台调度治理的完整业务闭环。

### 3.1 P1 角色与职责

| 角色 | 使用端 | 职责 |
|---|---|---|
| 游客/居民 | C 端小程序 | 下单、查看进度、联系服务人员、支付/价格异议、取消、确认完成、评价 |
| 便民服务人员 | B 端服务人员端 | 上下线、接单、到场报价、收款/凭证、开始服务、完工拍照、查看历史与收入 |
| 平台管理员 | Desktop | 服务概览、派单列表、片区、派单配置、服务人员、价格仲裁、取消审核、投诉与评价治理 |

### 3.2 C 端便民服务完整流程

| 子功能 | 需求 |
|---|---|
| 服务列表 | 展示可下单服务、说明、参考价/计量单位、点对点或片区式服务类型 |
| 一键下单 | 填写地址、终点地址、预约时间、描述、照片；夜间只支持预约；提交后自动进入订单详情 |
| 订单列表 | 仅展示便民服务订单，支持全部、进行中、已完成、已取消筛选 |
| 订单详情 | 展示订单号、服务类型、地址、描述、照片、服务人员、报价、状态进度 |
| 联系服务人员 | 已指派后可拨打服务人员电话 |
| 报价支付 | 服务人员报价后用户可线上支付；15 分钟未支付转人工处理 |
| 价格异议 | 用户对报价有异议时提交期望价格、原因、图片，进入平台价格仲裁 |
| 取消申请 | 未派单可直接取消；已指派/已接单/已核价走取消审批；服务中按规则限制取消 |
| 完工确认 | 服务人员完工后用户确认完成；超过时限可自动确认 |
| 评价 | 订单完成后评价服务与服务人员；影响服务人员诚信评分 |

### 3.3 B 端便民服务人员完整流程

| 子功能 | 需求 |
|---|---|
| 登录与定位 | 服务人员登录 B 端；只承接便民服务，不承接商城派单 |
| 工作台 | 显示在线状态、今日接单、累计接单、本月订单、进行中订单、最近订单 |
| 上下线 | 在线、休息、离线；忙碌由订单状态自动体现 |
| 新派单提醒 | 新订单弹窗倒计时，超时未响应自动转派或进入人工队列 |
| 任务列表 | 已指派、进行中、已完成分组；显示订单类型、地址、参考价、用户信息 |
| 接单/拒单 | 接单进入已接单；拒单进入重新派单或人工处理，并可影响诚信评分 |
| 到场报价 | 输入金额，显示参考价；超出参考价提示可能触发价格异议 |
| 等待支付 | 线上支付等待倒计时；超时转人工处理 |
| 线下收款 | 支持现金收款并上传支付凭证 |
| 开始服务 | 收款后进入服务中 |
| 完工拍照 | 至少上传 1 张带时间/位置水印的完工照片 |
| 历史订单 | 查看已完成/已取消订单、本月收入、现金/线上收入 |
| 个人中心 | 查看诚信评分、服务片区、资质证件、收入明细、帮助中心 |

### 3.4 Desktop 便民服务管理完整流程

便民服务管理必须保留以下下属菜单，且它们都是“便民服务订单派单”，不是商城派单。

| 菜单 | 子功能 |

| 派单列表 | 订单列表、状态筛选、自动派单流水、待人工处理、手动派单、重新尝试自动派单、取消申请处理 |

### 3.5 便民服务状态机

| 状态码 | 状态名称 | 触发来源 | 下一步 |
|---|---|---|---|
| S10 | 已下单 | C 端提交订单 | 自动进入待派单 |
| A10 | 待派单 | 系统进入派单池 | 自动派单或人工派单 |
| A20 | 已指派 | 系统/后台指派服务人员 | B 端接单或拒单 |
| A30 | 已接单 | B 端确认接单 | 到场报价 |
| A35 | 已核价 | B 端提交报价 | 用户支付或发起价格异议 |
| A38 | 协商中 | 用户发起价格异议 | 平台仲裁执行报价或取消 |
| A40 | 已收款 | 用户线上支付或服务人员现金收款 | 开始服务 |
| S48 | 服务中 | B 端开始服务 | 完工拍照 |
| S55 | 完工待确认 | B 端提交完工照片 | 用户确认或超时自动确认 |
| S40 | 已完成 | 用户确认/系统确认 | 评价、归档 |

| S50 | 已取消 | 用户取消/平台取消 | 归档 |
| S90 | 待人工处理 | 支付超时、协商超时、自动派单失败 | 后台重派或强制取消 |

### 3.6 异常与治理

| 场景 | 处理 |
|---|---|
| 自动派单失败 | 进入待人工处理，后台手动派单或重试 |
| 服务人员拒单 | 最多尝试 3 名服务人员，全部失败转人工；拒单可影响诚信评分 |
| 支付超时 | 15 分钟未支付转 S90 待人工处理 |
| 价格异议超时 | 30 分钟未处理转 S90 待人工处理 |
| 完工未确认 | 24 小时后自动确认完成 |
| 投诉成立 | 可触发服务人员诚信扣分 |
| 低诚信评分 | B 端提示限制经营，Desktop 可查看与调整 |

## 6. 一句话对外说明

本期新版重点不是重做所有老功能页面，而是在严格复用老版导游导览、资讯线路、商家/POI 信息维护、内容审核等成熟能力的基础上，补齐 C 端便民服务下单、B 端便民服务人员履约、Desktop 便民服务派单调度、投诉治理、诚信评分、供应商入驻、遗产知识和文化院落打卡等新版闭环。商城交易和商城派单统一移交商城后台及独立商城需求，当前平台后台不再承接商城订单履约。
